## 1. Dành cho Khách Hàng

### 1.1 Đăng nhập

1. Truy cập https://thue.eu.cc
2. Nhập **email** và **mật khẩu** đã được cấp
3. Bấm **Đăng nhập**
4. Nếu quên mật khẩu → liên hệ nhân viên qua Zalo hoặc hotline 0901 234 567

> **Lưu ý:** Mỗi tài khoản khách hàng do Admin tạo. Khách không tự đăng ký được.

### 1.2 Xem Dashboard

Sau khi đăng nhập, màn hình chính hiển thị:

- Tổng số đơn hàng
- Số đơn đang vận chuyển
- Số dư ví tiền
- Công nợ hiện tại
- Đơn hàng gần đây

### 1.3 Tạo đơn hàng mới

1. Bấm **Tạo đơn hàng** trên thanh menu hoặc dashboard
2. Điền thông tin:
   - **Link sản phẩm**: dán link từ Taobao, 1688, hoặc Tmall
   - **Số lượng**: nhập số lượng cần mua
   - **Ghi chú**: ghi thêm yêu cầu (màu sắc, size, v.v.)
3. Hệ thống **tự động tính phí** dựa trên:
   - Giá sản phẩm (CNY) × tỷ giá
   - Phí dịch vụ
   - Phí vận chuyển nội địa TQ
   - Phí vận chuyển quốc tế (theo kg)
   - Phí giao hàng VN
4. Bấm **Xác nhận** để gửi đơn
5. Nhân viên sẽ xử lý trong 1–2 giờ làm việc

### 1.4 Lưu ý khi gửi link Taobao / 1688 / Tmall

- Gửi link **đầy đủ** (copy từ thanh địa chỉ trình duyệt)
- Link phải là trang sản phẩm cụ thể, không phải trang tìm kiếm
- Ví dụ link hợp lệ: `https://item.taobao.com/item.htm?id=123456789`
- Ví dụ link hợp lệ: `https://detail.1688.com/offer/123456789.html`
- Nếu link không mở được → chụp ảnh sản phẩm + gửi kèm ghi chú

### 1.5 Theo dõi trạng thái đơn hàng

Vào **Đơn hàng** trên menu để xem danh sách đơn. Mỗi đơn có trạng thái:

| Trạng thái | Ý nghĩa |
|------------|---------|
| **Chờ mua** | Nhân viên chưa đặt hàng bên TQ |
| **Đã mua** | Đã đặt hàng, chờ người bán TQ gửi |
| **Người bán đã gửi** | Hàng đang gửi đến kho TQ |
| **Về kho TQ** | Hàng đã đến kho Trung Quốc |
| **Đang vận chuyển** | Hàng đang trên đường từ TQ về VN |
| **Về kho VN** | Hàng đã đến kho Việt Nam |
| **Đang giao** | Hàng đang được giao đến bạn |
| **Hoàn thành** | Bạn đã nhận hàng |
| **Đã huỷ** | Đơn bị huỷ |
| **Hoàn hàng** | Hàng đang được trả lại |

### 1.6 Xem chi tiết đơn hàng

Bấm vào bất kỳ đơn nào trong danh sách để xem:

- Mã đơn hàng (ví dụ: `BTH123456`)
- Link sản phẩm gốc
- Trạng thái hiện tại
- Cân nặng (khi có)
- Chi phí chi tiết
- Mã kiện / mã barcode
- Lịch sử trạng thái

### 1.7 Ví tiền

Vào **Ví tiền** trên menu:

- **Số dư**: tiền hiện có trong ví
- **Công nợ**: số tiền đang nợ (nếu có)
- **Lịch sử giao dịch**: nạp tiền, trừ tiền khi hoàn thành đơn, hoàn tiền khi huỷ đơn

### 1.8 Nạp tiền

1. Chuyển khoản đến tài khoản công ty:
   - **Ngân hàng:** Vietinbank CN Lạng Sơn
   - **Số tài khoản:** 110003049134
   - **Nội dung:** Ghi rõ **email đăng nhập** hoặc **mã khách hàng**
2. Liên hệ nhân viên để xác nhận (Zalo / hotline)
3. Admin sẽ cộng tiền vào ví cho bạn
4. Kiểm tra lại ở mục **Ví tiền** → **Lịch sử giao dịch**

### 1.9 Công nợ

- Khi đơn hoàn thành, hệ thống **tự động trừ tiền** từ ví
- Nếu ví không đủ → phần còn lại ghi vào **công nợ**
- Nạp tiền để giảm công nợ
- Công nợ cao có thể bị tạm ngưng dịch vụ

### 1.10 Giao hàng

- Khi hàng về kho Việt Nam, nhân viên liên hệ xác nhận địa chỉ giao
- Trạng thái chuyển sang **Đang giao**
- Sau khi nhận hàng → trạng thái chuyển sang **Hoàn thành**
- Nếu giao thất bại → nhân viên liên hệ lại để hẹn giao lần 2

### 1.11 Thông báo

Hệ thống gửi thông báo qua nhiều kênh:

- **App** (biểu tượng chuông trên web): thông báo trạng thái đơn, nạp tiền
- **Zalo OA**: nhận tin nhắn khi trạng thái đơn thay đổi (cần nhắn tin cho OA ít nhất 1 lần để liên kết)
- **Telegram**: nhận tin nhắn từ bot `@bactrunghai_bot`
- **Messenger**: nhắn tin cho page Facebook của công ty

### 1.12 Tra cứu đơn qua Chatbot

Bạn có thể gửi **mã đơn hàng** (ví dụ `BTH123456`) qua:

- **Zalo OA** → bot trả lời trạng thái + cân nặng + chi phí
- **Telegram** → gửi mã đơn trực tiếp trong chat
- **Messenger** → gửi mã đơn trong tin nhắn

Nếu không biết mã đơn → gửi câu hỏi bất kỳ, chatbot sẽ cố tìm câu trả lời phù hợp.

### 1.13 Lỗi thường gặp (Khách hàng)

| Vấn đề | Cách xử lý |
|--------|-----------|
| Không đăng nhập được | Kiểm tra email/mật khẩu, liên hệ admin nếu quên |
| Không thấy đơn cập nhật | Đơn cần thời gian xử lý, thử lại sau 1–2 giờ |
| Link sản phẩm không nhận | Kiểm tra link đầy đủ, thử copy lại từ trình duyệt |
| Ví tiền chưa được cộng | Liên hệ nhân viên xác nhận chuyển khoản |
| Chatbot không trả lời | Thử gửi lại, hoặc liên hệ hotline 0901 234 567 |
| Hàng chậm cập nhật | Có thể do tắc biên hoặc ngày lễ, liên hệ nhân viên |

## 2. Dành cho Admin

### 2.1 Đăng nhập Admin

1. Truy cập https://thue.eu.cc
2. Đăng nhập bằng tài khoản có role **ADMIN**
3. Hệ thống tự chuyển đến trang Admin

### 2.2 Dashboard Admin

Màn hình chính hiển thị:

- Tổng đơn hàng / đơn hôm nay
- Doanh thu / lợi nhuận
- Số khách hàng
- Đơn chờ xử lý
- Biểu đồ theo thời gian

### 2.3 Quản lý khách hàng

Vào **Quản lý người dùng**:

- Xem danh sách tất cả user (khách, admin, kho, kế toán)
- Tạo tài khoản mới cho khách hàng
- Chỉnh sửa thông tin (tên, email, số điện thoại, role)
- Xem số dư ví / công nợ của từng khách

### 2.4 Quản lý đơn hàng

#### Tạo đơn

1. Vào **Đơn hàng** → **Tạo đơn mới**
2. Chọn khách hàng
3. Nhập link sản phẩm, số lượng, ghi chú
4. Bấm **Xác nhận**

#### Xem / Sửa đơn

- Bấm vào đơn để xem chi tiết
- Có thể sửa: ghi chú, link sản phẩm, cân nặng, chi phí

#### Tìm kiếm đơn hàng

Thanh tìm kiếm hỗ trợ tìm theo:

- **Mã đơn hàng** (ví dụ: BTH123456)
- **Mã kiện** (package code)
- **Mã barcode**
- **Số điện thoại khách hàng**

#### Bộ lọc nhanh (Quick Filter Chips)

Bấm các chip lọc phía trên danh sách:

- **Có ghi chú**: đơn có order note
- **Có cập nhật khách hàng**: đơn có custom status note
- **Đang chờ lâu**: đơn PENDING ≥ 3 ngày
- **Đã huỷ**: đơn bị cancel
- **Hôm nay**: đơn tạo trong ngày
- **Chưa hoàn thành**: đơn chưa COMPLETED

Bấm **Xóa bộ lọc** để reset.

### 2.5 Cập nhật trạng thái đơn

#### Từ trang chi tiết đơn

1. Mở đơn hàng cần cập nhật
2. Trong card **Cập nhật trạng thái**, bấm nút trạng thái mới
3. Hệ thống tự validate chuyển đổi hợp lệ
4. Trạng thái mới được ghi vào audit log
5. Thông báo tự động gửi cho khách (nếu có Zalo/Telegram)

#### Quick Status Actions (từ danh sách)

- Cột **Thao tác** hiện nút chuyển trạng thái nhanh (ví dụ: "→ Đã mua")
- Bấm để chuyển trạng thái trực tiếp từ danh sách
- Không cần mở trang chi tiết

### 2.6 Status Summary Counters

Phía trên danh sách đơn có dãy chip đếm:

- **Chờ mua** (5) | **Đã mua** (3) | **Đang vận chuyển** (8) | **Tới kho VN** (2) | **Hoàn thành** (15)
- **Khẩn cấp** (1) — đếm đơn có priority URGENT

Bấm vào chip để lọc theo trạng thái đó.

### 2.7 Priority Tagging (Độ ưu tiên)

1. Mở chi tiết đơn hàng
2. Tìm card **Độ ưu tiên**
3. Chọn:
   - **Bình thường** (mặc định)
   - **Ưu tiên** (badge vàng)
   - **Khẩn cấp** (badge đỏ)
4. Đơn khẩn cấp hiện rõ trong danh sách và status counters

### 2.8 Custom Status Note (Ghi chú trạng thái)

1. Mở chi tiết đơn
2. Tìm card **Ghi chú trạng thái**
3. Nhập nội dung (ví dụ: "Đã liên hệ khách xác nhận địa chỉ")
4. Bấm cập nhật
5. Ghi chú hiện ở cột **Hoạt động** trong danh sách đơn

### 2.9 Order Notes

- Ghi chú nội bộ về đơn hàng
- Hiện ở cột **Hoạt động**: ai cập nhật cuối + thời gian (ví dụ "Admin • 5 phút trước")
- Đơn có ghi chú hiện **chấm xanh**, có custom status note hiện **chấm vàng**

### 2.10 Package Management (Quản lý kiện hàng)

- Xem danh sách kiện
- Tạo kiện mới (gom nhiều đơn vào 1 kiện)
- Xem ảnh kiện (nếu kho đã chụp)
- Xem barcode / mã kiện

### 2.11 Fee Settings (Cài đặt phí)

Vào **Cài đặt**:

- **Tỷ giá CNY → VND**
- **Phí dịch vụ (%)**
- **Phí vận chuyển nội địa TQ**
- **Phí vận chuyển quốc tế (VND/kg)**
- **Phí giao hàng VN**

Bấm **Lưu** để áp dụng cho đơn mới.

### 2.12 Notification Health (Trạng thái kênh thông báo)

Trên trang **Cài đặt**, card **Trạng thái kênh thông báo** hiện:

- Telegram: Bật/Tắt
- Zalo OA: Bật/Tắt
- Email: Bật/Tắt
- Messenger: Bật/Tắt
- App: Luôn bật

### 2.13 Support Knowledge Management (Trung tâm tri thức)

Vào **Trung tâm tri thức** trên sidebar:

#### Thêm mục tri thức mới

1. Bấm **+ Thêm mục mới**
2. Nhập:
   - **Tiêu đề**: câu hỏi khách thường hỏi
   - **Nội dung**: câu trả lời
   - **Danh mục**: chọn từ danh sách
   - **Từ khoá**: các từ khoá liên quan (cách nhau bằng dấu phẩy)
3. Bấm **Lưu**

#### Nhập hàng loạt

1. Bấm **Nhập hàng loạt**
2. Dán nội dung (dùng `#` hoặc `##` làm tiêu đề)
3. Chọn danh mục
4. Bấm **Nhập** — hệ thống tự tách thành nhiều mục

#### Tạo tri thức mẫu

Bấm **Tạo tri thức mẫu** để tạo 10 mục khởi đầu (giờ làm việc, liên hệ, tạo đơn, v.v.)

#### Thử câu hỏi chatbot

1. Bấm **Thử câu hỏi chatbot**
2. Nhập câu hỏi mẫu
3. Xem kết quả: matched/not matched, tiêu đề, nguồn match, điểm số
4. Nếu không match → thêm từ khoá hoặc mục tri thức mới

#### Sắp xếp tri thức

Dùng dropdown sắp xếp:

- **Mặc định**
- **Dùng nhiều nhất**: mục được chatbot dùng nhiều nhất
- **Mới sử dụng gần đây**: mục vừa match gần đây

Mỗi mục hiện: "Đã dùng X lần" (Z:/T:/M: theo kênh) + "Lần cuối: ..."

### 2.14 Unanswered Question Groups (Câu hỏi chưa trả lời)

Bấm **Câu hỏi chưa có câu trả lời** phía dưới trang Trung tâm tri thức:

#### Xem câu hỏi chưa trả lời

- Danh sách nhóm câu hỏi tương tự (gom theo nội dung)
- Mỗi nhóm hiện: kênh (Zalo/Telegram/Messenger), số lần hỏi, thời gian, danh mục

#### Bộ lọc

- **Tìm kiếm**: gõ từ khoá
- **Kênh**: Zalo / Telegram / Messenger / Tất cả
- **Trạng thái**: Chưa xử lý / Đã xử lý / Tất cả
- **Danh mục**: chọn danh mục hoặc "Chưa phân loại"

#### Quick Filter Chips

- **Tất cả**: hiện tất cả
- **Chưa có tri thức**: chỉ hiện nhóm chưa xử lý
- **Hỏi nhiều nhất**: sắp xếp theo số lần hỏi
- **Mới nhất**: sắp xếp theo thời gian gần nhất
- **Đã phân loại**: chỉ hiện nhóm đã gán danh mục

#### Xử lý câu hỏi

Với mỗi nhóm, có 3 thao tác:

1. **Gán danh mục**: chọn danh mục từ dropdown (Phí vận chuyển, Giờ làm việc, Nạp tiền, v.v.)
2. **+ Tạo tri thức**: mở form tạo tri thức mới, tự điền:
   - Tiêu đề = câu hỏi của khách
   - Nội dung = template "Khách hỏi: ... Trả lời:"
   - Từ khoá = tự trích từ câu hỏi
   - Danh mục = map từ danh mục nhóm
3. **Đã xử lý nhóm**: đánh dấu toàn bộ nhóm đã xử lý

### 2.15 Chatbot Analytics (Hiệu quả chatbot)

Bấm **Hiệu quả chatbot** trên trang Trung tâm tri thức:

- Tổng câu hỏi / Chưa xử lý / Đã xử lý
- Phân tích theo kênh (Zalo, Telegram, Messenger)
- Câu hỏi chưa xử lý gần nhất
- Top câu hỏi lặp lại nhiều nhất

### 2.16 Audit Log

Vào **Audit Log** trên sidebar:

- Lịch sử thay đổi trạng thái đơn
- Ai thay đổi, khi nào
- Từ trạng thái nào → sang trạng thái nào

### 2.17 Xử lý đơn bị lỗi / tắc

| Tình huống | Cách xử lý |
|-----------|-----------|
| Đơn PENDING quá lâu | Bấm chip "Đang chờ lâu" để lọc → liên hệ supplier TQ |
| Hàng mất tracking | Kiểm tra mã vận đơn TQ → liên hệ kho TQ |
| Khách gửi sai link | Ghi status note → liên hệ khách gửi lại |
| Hàng cấm / bị giữ | Cập nhật trạng thái CANCELLED → hoàn tiền ví cho khách |
| Giao hàng thất bại | Liên hệ khách xác nhận địa chỉ → giao lại |
| Hàng vỡ / hư | Chụp ảnh → ghi note → xử lý khiếu nại với kho TQ |

## 3. Kho Trung Quốc

### 3.1 Nhận hàng tại kho TQ

1. Đăng nhập bằng tài khoản role **WAREHOUSE_CN**
2. Vào **Dashboard kho TQ**
3. Bấm **Nhận hàng**

### 3.2 Nhập cân nặng

1. Đặt hàng lên cân
2. Nhập **cân nặng (kg)** vào form
3. Hệ thống tự tính phí vận chuyển quốc tế

### 3.3 Scan barcode / Package code

Có 2 cách nhập mã:

- **Máy quét USB**: quét barcode trên kiện, hệ thống tự nhận
- **Nhập tay**: gõ mã kiện vào ô nhập
- **Camera** (tuỳ chọn): bấm nút camera, đưa barcode vào khung hình

> **Lưu ý:** Camera scan cần HTTPS. Có cơ chế chống scan trùng (cooldown 3 giây).

### 3.4 Tạo kiện hàng

1. Vào **Tạo kiện**
2. Chọn các đơn hàng cần gom vào kiện
3. Nhập cân nặng tổng
4. Bấm **Tạo kiện** → hệ thống sinh mã kiện + barcode
5. In nhãn barcode (bấm nút In)

### 3.5 Cập nhật trạng thái

- Scan barcode hoặc nhập mã → hệ thống tự cập nhật trạng thái
- Trạng thái hợp lệ được validate bởi server
- Mỗi lần scan được ghi vào audit log

### 3.6 Lỗi thường gặp (Kho TQ)

| Vấn đề | Cách xử lý |
|--------|-----------|
| Scan không nhận barcode | Kiểm tra máy quét USB, thử nhập tay |
| Mã kiện không tìm thấy | Kiểm tra mã đúng chưa, thử nhập lại |
| Camera không mở | Cần HTTPS + cho phép truy cập camera trên trình duyệt |
| Trạng thái không chuyển được | Kiểm tra luồng trạng thái hợp lệ (không nhảy bước) |
| Cân nặng nhập sai | Sửa lại trong trang chi tiết kiện |

## 4. Kho Việt Nam

### 4.1 Nhận hàng về kho VN

1. Đăng nhập bằng tài khoản role **WAREHOUSE_VN**
2. Vào **Dashboard kho VN**
3. Bấm **Nhận hàng**
4. Scan hoặc nhập mã kiện
5. Hệ thống cập nhật trạng thái → **Về kho VN**

### 4.2 Scan kiện

- **Máy quét USB**: quét barcode trên kiện
- **Nhập tay**: gõ mã kiện
- **Camera**: quét bằng camera trình duyệt (cần HTTPS)

### 4.3 Dispatch / Giao hàng

1. Vào **Giao hàng**
2. Chọn kiện cần giao
3. Bấm **Xuất kho** → trạng thái chuyển sang **Đang giao**
4. Thông báo tự động gửi cho khách

### 4.4 Hoàn tất giao hàng

1. Sau khi giao thành công → bấm **Hoàn thành**
2. Trạng thái chuyển sang **Hoàn thành**
3. Hệ thống tự trừ tiền từ ví khách
4. Nếu ví không đủ → ghi vào công nợ

### 4.5 Xử lý hàng chưa đủ thông tin

- Kiện không có mã đơn → liên hệ Admin kiểm tra
- Kiện không rõ khách → ghi chú + để riêng + báo Admin
- Kiện bị hư hỏng → chụp ảnh + ghi chú + báo Admin

### 4.6 Lỗi thường gặp (Kho VN)

| Vấn đề | Cách xử lý |
|--------|-----------|
| Scan không nhận | Kiểm tra máy quét, thử nhập tay |
| Kiện không có trong hệ thống | Có thể kho TQ chưa tạo kiện → liên hệ kho TQ |
| Giao hàng thất bại | Ghi chú lý do → liên hệ khách hẹn giao lại |
| Không tìm thấy khách | Kiểm tra mã đơn → tìm khách qua Admin |

  - `/status` — hướng dẫn tra cứu đơn

- Gửi mã đơn → trả về trạng thái

### 6.4 Cách chatbot dùng SupportKnowledge

Khi khách gửi tin nhắn (không phải mã đơn):

1. Chatbot tìm trong bảng **SupportKnowledge** (mục đang bật)
2. So khớp theo thứ tự ưu tiên:
   - **Từ khoá** (ưu tiên cao nhất)
   - **Tiêu đề**
   - **Danh mục**
   - **Nội dung**
3. Nếu tìm thấy → trả lời nội dung tri thức
4. Nếu không tìm thấy → trả lời hướng dẫn chung + ghi vào danh sách "Câu hỏi chưa trả lời"

### 6.5 Cách thêm tri thức mới

**Cách 1: Thêm thủ công**
1. Vào Admin → Trung tâm tri thức → + Thêm mục mới
2. Nhập tiêu đề, nội dung, danh mục, từ khoá

**Cách 2: Từ câu hỏi chưa trả lời**
1. Vào Câu hỏi chưa trả lời
2. Bấm **+ Tạo tri thức** trên nhóm cần xử lý
3. Form tự điền tiêu đề + từ khoá → chỉ cần viết câu trả lời

**Cách 3: Nhập hàng loạt**
1. Bấm **Nhập hàng loạt**
2. Dán nội dung văn bản (dùng tiêu đề `#` để tách mục)

**Cách 4: Chạy script seed**
```bash
npm run seed:knowledge
```
Tự nhập 139 mục tri thức có sẵn (an toàn, không trùng lặp).

### 6.6 Cách xử lý câu hỏi chưa trả lời

1. Mở **Câu hỏi chưa trả lời** → xem danh sách
2. Dùng **Quick Filters** để tìm nhanh:
   - "Hỏi nhiều nhất" → xử lý câu lặp lại nhiều trước
   - "Chưa có tri thức" → câu chưa ai xử lý
3. Với mỗi nhóm:
   - Gán danh mục (nếu chưa có)
   - Bấm **+ Tạo tri thức** → viết câu trả lời
   - Hoặc bấm **Đã xử lý nhóm** nếu đã có tri thức tương tự

### 6.7 Cách theo dõi câu hỏi lặp lại

- Hệ thống tự nhóm câu hỏi tương tự (case-insensitive)
- Badge "X lần" cho biết tần suất hỏi
- Dùng chip **Hỏi nhiều nhất** để sắp xếp
- Dùng **Hiệu quả chatbot** để xem top câu hỏi lặp

### 6.8 Cách viết câu trả lời tốt cho chatbot

| Nên | Không nên |
|-----|----------|
| Ngắn gọn, dễ hiểu | Dài dòng, chuyên ngành |
| Giọng nhân viên CSKH thật | Giọng robot, máy móc |
| Có ví dụ cụ thể (mã đơn BTH...) | Nói chung chung |
| Thêm từ khoá phù hợp | Để trống từ khoá |
| Phù hợp đọc trên điện thoại | Dùng bảng/markdown phức tạp |
| Tiếng Việt tự nhiên | Tiếng Anh hoặc Trung |

**Ví dụ câu trả lời tốt:**

> Hàng về kho TQ nghĩa là đơn đã đến kho bên Trung Quốc. Nhân viên sẽ kiểm tra, cân nặng và đóng kiện. Sau đó hàng được chuyển về Việt Nam, thường mất 3–5 ngày. Quý khách cứ yên tâm theo dõi trạng thái trên hệ thống nhé!

## 7. Vận Hành Hệ Thống

### 7.1 PM2 Production

App chạy trên Windows Server với PM2:

```bash
# Xem trạng thái
pm2 status

# Xem log realtime
pm2 logs logistics-system

# Xem log 100 dòng gần nhất
pm2 logs logistics-system --lines 100

# Xem log lỗi
pm2 logs logistics-system --err

# Monitoring dashboard
pm2 monit
```

### 7.2 Restart App

```bash
# Restart bình thường
pm2 restart logistics-system

# Restart sau khi cập nhật code (full rebuild)
git pull origin main
npm install
npx prisma generate
npm run build
pm2 restart logistics-system
```

### 7.3 Pull Latest Main

```bash
cd logistics-system
git pull origin main
```

### 7.4 Migrate Deploy

Khi có migration mới (thay đổi database schema):

```bash
npx prisma migrate deploy
```

> **Lưu ý:** Luôn backup database trước khi chạy migration trên production.

### 7.5 Prisma Generate

Khi schema.prisma thay đổi:

```bash
npx prisma generate
```

### 7.6 Build

```bash
npm run build
```

Nếu build lỗi → kiểm tra `npm run typecheck` trước để tìm lỗi TypeScript.

### 7.7 Seed Knowledge (Nhập tri thức)

```bash
# Chỉ nhập SupportKnowledge (an toàn cho production)
npm run seed:knowledge

# Nhập toàn bộ demo data (CHỈ dùng cho dev/test)
npm run seed
```

### 7.8 Kiểm tra PostgreSQL Docker

```bash
# Xem container đang chạy
docker ps | grep postgres

# Nếu không chạy → khởi động lại
docker start logistics-postgres

# Kiểm tra kết nối
docker exec logistics-postgres pg_isready -U postgres
```

### 7.9 Database Backup

```bash
# Backup
docker exec logistics-postgres pg_dump -U postgres logistics_db > backup_$(date +%Y%m%d).sql

# Restore
docker exec -i logistics-postgres psql -U postgres logistics_db < backup.sql
```

### 7.10 Xử lý lỗi đăng nhập

| Tình huống | Cách xử lý |
|-----------|-----------|
| Tất cả user không đăng nhập được | Kiểm tra app có đang chạy: `pm2 status` |
| Lỗi "Invalid credentials" | Kiểm tra email/password đúng chưa |
| Lỗi database connection | Kiểm tra PostgreSQL: `docker ps` |
| Session expired | Đăng nhập lại bình thường |
| NEXTAUTH_SECRET thay đổi | Tất cả session cũ sẽ hết hạn, user cần đăng nhập lại |

### 7.11 Xử lý app mất kết nối

1. Kiểm tra PM2: `pm2 status`
2. Nếu app stopped → `pm2 restart logistics-system`
3. Kiểm tra log lỗi: `pm2 logs logistics-system --err`
4. Kiểm tra database: `docker ps | grep postgres`
5. Nếu DB stopped → `docker start logistics-postgres`
6. Kiểm tra network/DNS → `curl -I https://thue.eu.cc`
7. Nếu vẫn lỗi → xem log chi tiết: `pm2 logs logistics-system --lines 200`
